# 02 Being Clear and Direct

## 核心理念
- Claude 不知道你的隐含意图，想要什么就要直说。
- 提示越清晰、约束越具体，输出越稳定。

## 教程内容
- 用“同事看得懂，模型才看得懂”定义清晰 prompt。
- 可直接要求“跳过前言”，减少废话。
- 可直接要求“只给答案”，避免模糊表态。
- 主观问题如果想要单一结论，就必须显式施压。
- 长输出通常要把目标字数写得更高一些。

## 练习内容
- `Exercise 2.1 - Spanish`：让模型强制用西班牙语回答。
- `Exercise 2.2 - One Player Only`：只输出一个球员名字，不能带多余内容。
- `Exercise 2.3 - Write a Story`：尽量逼出超过 800 词的长故事。
