"invalid use of non-static member function" 错误通常发生在尝试在没有对象的情况下调用类的非静态成员函数时。非静态成员函数需要一个对象实例来访问类的成员变量和其他非静态成员函数，因为它们隐含地依赖于`this`指针。


**内存泄漏**指程序通过 `new`/`malloc` 等操作申请内存后，没有通过 `delete`/`free` 释放内存。长期运行的程序（如服务器）会逐渐耗尽内存，最终导致性能下降或崩溃。

**C风格字符串的遗留影响**：C语言中，字符串以 `'\0'` 结尾，例如 `"abc"` 的实际存储是 `{'a', 'b', 'c', '\0'}`，因此 `s[3]` 是 `'\0'`。
**但 `std::string` 不同**：它的 `size()` 仅统计有效字符，不包含末尾的 `'\0'`（除非手动添加）
对于`string str = "awa"; char c = str[3]`是越界访问
