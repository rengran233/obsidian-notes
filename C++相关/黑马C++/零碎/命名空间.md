

在C++中，**命名空间（Namespace）** 是一种组织代码和避免命名冲突的重要机制，它通过作用域隔离让程序结构更清晰。

---

### **1. 命名空间的核心作用**
- **避免名称冲突**：  
  不同模块或库可以使用相同名称的标识符（如函数、类），只要它们位于不同的命名空间中。  
  ```cpp
  namespace LibraryA {
      void print() { /* ... */ }
  }
  namespace LibraryB {
      void print() { /* ... */ }
  }
  int main() {
      LibraryA::print();  // 调用A的版本
      LibraryB::print();  // 调用B的版本
      return 0;
  }
  ```

- **封装代码逻辑**：  
  将相关功能集中在一个命名空间内，提高代码可读性和模块化。例如，数学工具函数可以放在 `MathUtils` 命名空间中。

---

### **2. 命名空间的核心特性**
- **作用域统一性**：  
  命名空间内的所有标识符共享同一作用域，需通过 `命名空间::标识符` 访问（选项B正确）。  
  ```cpp
  namespace MySpace {
      int value = 42;
      void func() { /* ... */ }
  }
  // 访问方式：
  MySpace::value = 10;
  MySpace::func();
  ```

- **支持嵌套和扩展**：  
  - **嵌套命名空间**：  
    ```cpp
    namespace Project {
        namespace Graphics {
            class Renderer { /* ... */ };
        }
    }
    // 访问方式：Project::Graphics::Renderer
    ```
  - **命名空间扩展**：  
    可以分多次向同一命名空间添加内容（适用于大型项目）：  
    ```cpp
    namespace MyLib {
        void func1();
    }
    namespace MyLib {  // 扩展命名空间
        void func2();
    }
    ```

- **别名支持**：  
  简化长命名空间的访问：  
  ```cpp
  namespace VeryLongNamespaceName { /* ... */ }
  namespace VLN = VeryLongNamespaceName;  // 别名
  VLN::func();
  ```

---

### **3. 匿名命名空间**
- **作用**：  
  定义在匿名命名空间中的标识符仅在当前文件内可见（类似 `static` 关键字，但更现代）。  
  ```cpp
  namespace {  // 匿名命名空间
      int internalVar;  // 仅在当前编译单元可见
  }
  ```

---

### **4. 实际应用场景**
- **多人协作开发**：  
  团队中每个成员负责的模块可置于独立命名空间，避免函数或类名冲突。
- **第三方库封装**：  
  如 `std` 是C++标准库的命名空间，第三方库（如 `Boost`）也通过命名空间隔离代码。
- **版本控制**：  
  不同版本的代码可放置在不同命名空间中（如 `v1::API` 和 `v2::API`）。
