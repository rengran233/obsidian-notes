这三个概念是前端组件设计（尤其是编写通用组件库，如 Ant Design, Element Plus）时的核心架构思想。

它们的本质区别在于：**“谁掌握着数据的真理（Source of Truth）？”** 是父组件（使用者），还是子组件（自己）？

我们可以把组件比作一个“木偶”：

1. **受控组件**：提线木偶。完全由父组件控制，动不动全看父组件。
    
2. **非受控组件**：发条玩具。父组件给它上个发条（初始值），剩下的它自己动。
    
3. **混合组件**：智能机器人。既能听指令（受控），也能自己动（非受控）。
    

---

### 1. 受控组件 (Controlled Component)

**真理掌握在父组件手中。**

组件内部**没有任何**自己的状态（State）。它只做两件事：

1. **读**：通过 `props` 接收当前要显示的值。
    
2. **写**：当用户交互时，通过 `emit` 通知父组件“我想改值”。
    

**如果父组件不更新 props，子组件在页面上就永远不会变。**

#### 代码示例 (Vue 3)

Code snippet

```
<template>
  <input 
    :value="modelValue" 
    @input="$emit('update:modelValue', $event.target.value)"
  />
</template>

<script setup>
defineProps(['modelValue']); // 只接收，不保留
defineEmits(['update:modelValue']);
</script>
```

#### 特点

- ✅ **优点**：单一数据源（Single Source of Truth），父组件完全掌控 UI，方便做表单校验、即时格式化（比如输入小写自动变大写）。
    
- ❌ **缺点**：如果页面很简单，父组件还得写一堆 `v-model` 或 state 变量，比较繁琐。
    

---

### 2. 非受控组件 (Uncontrolled Component)

**真理掌握在子组件自己手中。**

组件内部维护一个 `ref` 或 `data`。父组件只负责传递一个**初始值 (Initial Value)**，之后组件内部的变化与父组件无关。

如果父组件想要拿到当前的值，通常需要通过 **Ref** 或者 **Form Submit** 事件。

#### 代码示例 (Vue 3)

Code snippet

```
<template>
  <input v-model="internalValue" />
</template>

<script setup>
import { ref } from 'vue';

const props = defineProps(['defaultValue']); // 只接收初始值

// 复制一份做成内部状态，之后就跟 props 没关系了
const internalValue = ref(props.defaultValue || ''); 

// 暴露给父组件，让父组件通过 ref 读取
defineExpose({ getValue: () => internalValue.value });
</script>
```

#### 特点

- ✅ **优点**：简单，父组件用起来省心，不需要维护 state，即插即用。
    
- ❌ **缺点**：父组件很难干预。比如父组件想“重置”这个输入框，或者想实时校验，通过 Ref 操作 DOM/实例会很麻烦且不优雅。

### 3. 混合组件 (Hybrid / Semi-Controlled) —— 终极形态

这是所有成熟组件库（如 Element Plus, Vant）采用的模式。

**设计哲学：**

> "如果你传了 `modelValue` (受控属性)，我就听你的；
> 
> 如果你没传，或者只传了 `defaultValue`，我就自己管理状态。"

它结合了前两者的优点：既可以被深度控制，也可以拿来即用。

#### 实现原理：内外同步

组件内部维护一个 `innerValue`。

1. **Watch Props**: 当父组件的 `modelValue` 变化时，强制更新 `innerValue`。
    
2. **Emit Event**: 当用户操作时，通知父组件更新，**同时**暂时更新 `innerValue` (为了 UI 响应速度)。
    

#### 代码示例 (Vue 3 手写实现逻辑)

_注：Vue 3.4+ 提供了 `defineModel` 宏，完美简化了这个模式，但为了理解原理，我们看手动实现版。_

Code snippet

```
<template>
  <input :value="innerValue" @input="handleInput" />
</template>

<script setup>
import { ref, watch, onMounted } from 'vue';

const props = defineProps({
  modelValue: { type: String, default: undefined }, // 受控属性
  defaultValue: { type: String, default: '' }       // 非受控初始值
});

const emit = defineEmits(['update:modelValue', 'change']);

// 1. 内部状态：优先取 modelValue，没有才取 defaultValue
const innerValue = ref(
  props.modelValue !== undefined ? props.modelValue : props.defaultValue
);

// 2. 监听外部变化（受控模式下，父组件变了，内部也要变）
watch(() => props.modelValue, (newVal) => {
  if (newVal !== undefined) {
    innerValue.value = newVal;
  }
});

// 3. 处理用户输入
const handleInput = (e) => {
  const value = e.target.value;
  
  // 更新内部状态（让 UI 动起来）
  // 注意：如果是严格受控，这里其实可以不更新，等父组件流回来。
  // 但为了体验通常会直接更新，或者判断 props.modelValue === undefined 才更新
  innerValue.value = value;
  
  // 通知父组件
  emit('update:modelValue', value);
  emit('change', value);
};
</script>
```

#### 父组件怎么用它？

**用法 A：受控模式 (想实时控制)**

HTML

```
<HybridInput v-model="myData" /> 
```

**用法 B：非受控模式 (懒人模式)**

HTML

```
<HybridInput default-value="Hello" />
```

### 总结对比表

|**特性**|**受控组件 (Controlled)**|**非受控组件 (Uncontrolled)**|**混合组件 (Hybrid)**|
|---|---|---|---|
|**数据源**|Props|内部 State|Props (优先) 或 内部 State|
|**初始值**|Props|Props (作为初始值)|判断 Props 是否存在|
|**更新方式**|父组件修改 Props|组件自己修改 State|尝试修改 State 并通知父组件|
|**适用场景**|复杂表单、实时校验|简单的展示型组件、文件上传|**通用组件库** (Input, Select, Tabs)|
|**复杂度**|低 (逻辑在父组件)|中|高 (需要同步逻辑)|

### 什么时候用哪个？

1. **写业务页面时**：通常直接写 **受控组件** (v-model)。因为业务逻辑通常在父组件，且 Vue 的 v-model 语法糖让受控并不麻烦。
    
2. **封装简单组件时**：如果只是为了封装样式（比如一个带样式的 div），不需要交互，可以是无状态的。
    
3. **造轮子（开发组件库）时**：**必须使用混合组件模式**。你必须假设用户既可能想要 `v-model` 双向绑定，也可能只想写个 `<MyTabs default-active="1" />` 就不管了。
