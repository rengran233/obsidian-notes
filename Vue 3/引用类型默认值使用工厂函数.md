这是一个非常经典的“防坑”设定，它的根本原因不在于 Vue 本身，而在于 **JavaScript 对引用类型（对象和数组）的内存分配机制**。

一句话总结：**为了防止多个组件实例共享同一份默认数据，导致状态互相污染（数据泄漏）。**

---

### 1. 灾难现场：如果不使用工厂函数会怎样？

在 JavaScript 中，基本数据类型（如 `String`, `Number`, `Boolean`）是**按值传递**的，而对象和数组是**按引用传递**的。

如果 Vue 允许你直接把一个对象字面量写成默认值：

```javascript
// ❌ 假设 Vue 允许这种错误的写法
props: {
  userList: {
    type: Array,
    default: [] // 内存中只创建了这一个单独的空数组 [内存地址: 0x001]
  }
}

```

当你把这个组件在页面上使用了 3 次（即生成了 3 个组件实例），且都没有给它们传 `userList` 时：

1. 组件 A 使用了默认值（指向内存 `0x001`）。
2. 组件 B 使用了默认值（指向内存 `0x001`）。
3. 组件 C 使用了默认值（指向内存 `0x001`）。

这个时候，如果**组件 A** 执行了 `this.userList.push('张三')`，因为它们共享同一个内存地址，**组件 B 和 组件 C 的 `userList` 也会凭空多出一个“张三”**！这种组件间的幽灵联动，会导致极其难以排查的 Bug。

### 2. 拯救方案：工厂函数的作用

为了保证每个组件实例都有自己**独立**的默认数据，Vue 强制要求对象和数组的默认值必须是一个**工厂函数**（Factory Function）。

```javascript
// ✅ 正确的写法
props: {
  userList: {
    type: Array,
    default: () => [] // 这是一个函数
  }
}

```

当你使用了工厂函数后，Vue 在初始化每个组件实例时，**都会调用一次这个 `default` 函数**。

* 组件 A 初始化：调用 `default()`，返回一个**全新的** `[]`（内存地址 `0x001`）。
* 组件 B 初始化：调用 `default()`，返回一个**全新的** `[]`（内存地址 `0x002`）。
* 组件 C 初始化：调用 `default()`，返回一个**全新的** `[]`（内存地址 `0x003`）。

现在，无论你在组件 A 里怎么 `push` 数据，都绝不会影响到 B 和 C，因为它们的内存地址是完全隔绝的。

---

### 3. 补充：为什么 String/Number/Boolean 不需要？

因为基础数据类型在内存中不可变且按值复制。

```javascript
props: {
  count: {
    type: Number,
    default: 0
  }
}

```

即便三个组件都使用了默认值 `0`，当组件 A 把 `count` 改为 `1` 时，它只是在自己的作用域里把变量指向了新值 `1`，完全不会影响其他组件里的 `0`。
