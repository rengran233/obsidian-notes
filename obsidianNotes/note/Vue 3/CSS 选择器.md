CSS 选择器是前端开发的“定位系统”。作为一名在改组件库的实习生，你遇到的**90% 的样式修改无效问题**，通常都是因为**选择器权重（Specificity）** 或者 **作用域（Scope）** 没搞清楚。

我将 CSS 选择器按**实用程度**和**组件库开发场景**为你分类整理：

---

### 1. 基础选择器 (最常用)

这是构建组件样式的基石，你应该最熟悉：

| 选择器 | 写法 | 示例 | 说明 |
| --- | --- | --- | --- |
| **通配符** | `*` | `* { margin: 0; }` | 选中所有元素（通常用于重置样式）。 |
| **标签** | `tag` | `div { color: red; }` | 选中所有该标签（在组件库中很少直接用，容易污染全局）。 |
| **类** | `.class` | `.btn { ... }` | **最核心**。组件库通常使用 BEM 命名法（如 `.el-button__icon`）。 |
| **ID** | `#id` | `#app { ... }` | 选中唯一元素。**组件库开发中绝对禁止使用**，因为组件会被复用，ID 必须唯一。 |

---

### 2. 关系选择器 (组件结构的逻辑)

当你需要控制“组件内部的子元素”时，这些选择器非常关键。

* **后代选择器 (`space`)**
* 写法：`.card .title` (中间有个空格)
* 作用：选中 `.card` 内部**所有**的 `.title`（不管是儿子还是孙子）。
* *注意：层级太深会增加渲染成本，尽量保持扁平。*


* **子代选择器 (`>`)**
* 写法：`.list > .item`
* 作用：只选中**亲儿子**。
* *场景：防止样式污染到嵌套在里面的其他组件。*


* **相邻兄弟选择器 (`+`)**
* 写法：`.label + .input`
* 作用：选中紧跟在 `.label` 后面的那个 `.input`。
* *场景：表单布局中，给相邻的两个输入框之间加间距。*


* **通用兄弟选择器 (`~`)**
* 写法：`.checkbox:checked ~ .content`
* 作用：选中 `.checkbox` 后面所有的 `.content` 兄弟。



---

### 3. 属性选择器 (组件库神器)

在组件库（尤其是表单组件）中，属性选择器非常好用，因为它能根据 HTML 的属性直接定样式，不需要额外的类名。

| 写法 | 示例 | 场景 |
| --- | --- | --- |
| `[attr]` | `[disabled]` | 选中所有禁用的元素（置灰、鼠标禁止）。 |
| `[attr=val]` | `input[type="text"]` | 精准选中某种类型的输入框。 |
| `[attr*=val]` | `a[href*="login"]` | 选中 href 属性中**包含** "login" 的链接。 |
| `[attr^=val]` | `div[class^="el-icon-"]` | 选中所有类名以 `el-icon-` **开头**的元素（用于图标字体库）。 |

---

### 4. 伪类选择器 (状态与逻辑)

组件的“交互感”全靠它。

* **交互状态**：
* `:hover` (鼠标悬停)
* `:active` (点击按下时)
* `:focus` (获得焦点，如输入框输入时)
* `:visited` (链接已访问)


* **结构逻辑**：
* `:first-child` / `:last-child`：第一个/最后一个。
* `:nth-child(n)`：第 n 个（可以写 `2n` 表示偶数，`odd` 表示奇数）。
* *场景：斑马纹表格 (Striped Table)*


* `:not(selector)`：**排除法**。
* *场景：`.btn:not(.disabled)` —— 给所有没禁用的按钮加 hover 效果。*





---

### 5. 伪元素选择器 (装饰与修饰)

它们不是真正的 HTML 标签，而是 CSS 创建的“假元素”。（注意是双冒号 `::`，虽然单冒号也能跑，但双冒号是规范）。

* `::before` / `::after`：
* 在元素内容的前面/后面插入内容。
* *场景：给按钮加小图标、清除浮动 (clearfix)、Tooltip 的小三角。*


* `::placeholder`：
* 修改 Input 占位符的颜色。



---

### 实习生特别篇：Vue 深度选择器 (Deep Selector)

这是你在改组件库时**必须掌握**的知识点。

在 Vue 中，为了防止样式污染，我们通常会在 `<style>` 标签上加 `scoped` 属性。但这带来了一个问题：**父组件的样式无法渗透到子组件内部。**

**场景**：你在用 Element Plus 的 `el-input`，你想改它内部那个 `<input>` 标签的背景色，直接写 CSS 会无效。

**解决方法（样式穿透）：**

1. **Vue 2 / 旧写法**：
```css
.parent >>> .child { ... }
/* 或者 */
.parent /deep/ .child { ... }

```


2. **Vue 3 (推荐)**：
使用 `:deep()` 函数。
```html
<style scoped>
.my-wrapper :deep(.el-input__inner) {
  background-color: #f0f0f0;
}
</style>

```



### 总结：选择器权重 (排位赛)

当你发现写了样式不生效，通常是被“权重更高”的选择器覆盖了。记住这个口诀：

**`!important` > 行内样式 > ID选择器 > 类选择器/属性/伪类 > 标签选择器 > 通配符**

* **ID**: 100 分
* **Class**: 10 分
* **Tag**: 1 分

**建议**：作为实习生，修改组件样式时，**尽量不要用 `!important**`，因为这会让后续维护变得极其痛苦。尝试通过增加父级类名（提高权重）来覆盖样式。
