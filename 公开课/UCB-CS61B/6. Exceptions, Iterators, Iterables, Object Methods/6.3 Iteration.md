The Java built-in collections support enhanced `for` loop because they inherit `Iterable<>` interface, we can implements iterator for our custom class to let them support it.
```Java
import java.util.Iterator;  
  
// use Iterable as a sign to support enhanced for loop
// implement iterator() and specific iterator class
public class ASet<T> implements Iterable<T> {  
    private T[] items;  
    private int size;
	...
	@Override
	// return Iterator<T> instead of custom iterator 
	// whose visibility scope is within ASet class
    public Iterator<T> iterator() {
        return new ASetIterator();  
    }  
  
    // implements Iterator for a specific iterator 
    // implement next() and hasNext()
    public class ASetIterator implements Iterator<T> {  
        private int pos;  
  
        public ASetIterator() {  
            pos = 0;  
        }  
  
		@Override
        public T next() {  
            T res = items[pos];  
            pos += 1;  
            return res;  
        }  
  
		@Override
        public boolean hasNext() {  
            return pos < size;  
        }  
    }  
}
```
