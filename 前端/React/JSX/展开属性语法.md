可以用`<Input {...obj} />`的语法来将obj中的属性逐一传递给组件，等价于`<Input prop1={p1}  />`，对于obj的属性不固定时为必选

这份关于 **JSX 展开属性 (JSX Spread Attributes)** 的简要笔记，适合复习和答辩备忘：

#### 1. 核心语法

* **写法：** `<Component {...obj} />`
* **含义：** 将对象 `obj` 中的**所有可枚举属性**，一次性作为 `props` 传递给组件。

#### 2. 代码对照

假设有一个配置对象：
```javascript
const config = {
  placeholder: "请输入...",
  disabled: true,
  size: "large"
};
```
**使用展开属性（推荐）：**
```jsx
<Input {...config} />
```
**等价于（传统写法）：**
```jsx
<Input 
  placeholder={config.placeholder}
  disabled={config.disabled}
  size={config.size}
/>
```

#### 3. 核心优势

1. **动态性 (Dynamic):** 前端不需要知道对象里具体有哪些属性。如果后端新增了配置（如 `maxLength`），组件会自动接收并生效，无需修改前端代码。
2. **简洁性 (Concise):** 避免了重复编写 `prop={obj.prop}` 的冗余代码。
3. **解耦 (Decoupling):** 实现了**Props 透传**。父组件（如 `AdModal`）只负责传递，子组件（如 `Input`）负责消费，中间层不需要关心具体逻辑。

#### 4. ⚠️ 关键注意点：覆盖顺序

属性的顺序决定了谁覆盖谁（后面的覆盖前面的）。

* **场景 A：默认值被覆盖**
```jsx
// 如果 config 里有 className，会覆盖前面的 'default-class'
<div className="default-class" {...config} />
```

* **场景 B：强制覆盖（常用）**
```jsx
// 无论 config 里有没有 disabled，这里都强制为 true
<Input {...config} disabled={true} />
```

#### 5. 项目中的应用（答辩话术）

> “在本项目中，我利用 JSX 展开属性实现了**后端驱动 UI** 的动态配置透传。前端通过 `{...field.props}` 将后端下发的 Schema 配置直接传递给 Ant Design 组件，这使得系统具备了极高的扩展性，后端调整 UI 细节时前端无需发版。”
