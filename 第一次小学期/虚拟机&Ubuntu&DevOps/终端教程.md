### **Linux 终端快速入门实践教程**

#### **核心概念**

1.  **终端 (Terminal)**: 你输入命令的那个黑色窗口。
2.  **Shell**: 真正**解释和执行**你命令的程序。你输入命令给终端，终端把它交给 Shell，Shell 执行完把结果返回给终端显示。我们用的通常是 `Bash` (Bourne Again SHell)。
3.  **命令 (Command)**: 你让计算机做某件事的指令，通常结构是：`命令 [选项] [参数]`。
    *   `命令`: 动词，如 `ls` (列出), `cd` (切换目录)。
    *   `选项`: 通常以 `-` 或 `--` 开头，用来改变命令的行为，如 `ls -l` (用列表形式显示)。
    *   `参数`: 命令作用的对象，如 `ls /home` (列出 /home 目录下的内容)。

#### **一、 文件和目录操作 (最最基础)**

这是你每天都要用到的。打开你的终端，我们现在就开始实践！

1.  **`pwd` (Print Working Directory) - 我在哪？**
    *   在终端输入 `pwd` 然后回车。它会显示你当前所在的目录路径。
    *   刚打开终端时，你通常在你的“家目录”下，比如 `/home/rengran233`。

2.  **`ls` (List) - 这里有什么？**
    *   输入 `ls`。它会列出当前目录下的所有文件和文件夹。
    *   **常用组合：**
        *   `ls -l`: 以**长列表**形式显示，包含权限、所有者、大小、修改日期等详细信息。(**l** for **l**ong)
        *   `ls -a`: 显示所有文件，包括以 `.` 开头的**隐藏文件**。( **a** for **a**ll)
        *   `ls -lh`: `l` 和 `h` 选项的组合，以列表形式显示，并且文件大小会以 K, M, G 等**人类可读 (h**uman-readable) 的格式显示。**（强烈推荐，日常使用最多）**
        *   `ls -R`: 递归显示所有子目录的内容。( **R** for **R**ecursive)

3.  **`cd` (Change Directory) - 我要去...**
    *   `cd Desktop`: 进入当前目录下的 `Desktop` 文件夹 (如果存在的话)。
    *   `cd /var/log`: 切换到根目录下的 `/var/log` 目录（绝对路径）。
    *   `cd ..`: 返回到**上一级**目录。
    *   `cd` (后面不跟任何东西): 快速返回到你的**家目录**。
    *   `cd ~`: 和 `cd` 效果一样，`~` 代表家目录。
    *   `cd -`: 返回到你**上一次所在**的目录，非常方便！

4.  **`mkdir` (Make Directory) - 创建一个文件夹**
    *   `mkdir devops-projects`: 在当前位置创建一个名为 `devops-projects` 的文件夹。

5.  **`touch` (Touch) - 创建一个空文件**
    *   `touch readme.md`: 在当前位置创建一个名为 `readme.md` 的空文件。

6.  **`cp` (Copy) - 复制**
    *   `cp readme.md backup.md`: 把 `readme.md` 复制一份，命名为 `backup.md`。
    *   `cp readme.md devops-projects/`: 把 `readme.md` 复制到 `devops-projects` 文件夹里。
    *   `cp -r devops-projects/ devops-backup/`: 复制整个文件夹，**必须加 `-r`** (recursive)。

7.  **`mv` (Move) - 移动或重命名**
    *   **移动**: `mv backup.md devops-projects/` - 把 `backup.md` 移动到 `devops-projects` 文件夹。
    *   **重命名**: `mv readme.md introduction.md` - 把 `readme.md` 重命名为 `introduction.md`。

8.  **`rm` (Remove) - 删除**
    *   **警告：Linux 的 `rm` 没有回收站！删除后无法恢复！请务必小心！**
    *   `rm introduction.md`: 删除 `introduction.md` 文件。
    *   `rm -r devops-backup/`: 删除整个文件夹和里面的所有内容，**必须加 `-r`**。
    *   **高危命令**: `rm -rf /` 是传说中删库跑路的命令，**千万不要尝试！** (`-f` 是 force，强制删除不提示)。

#### **二、 查看文件内容**

1.  **`cat` (Concatenate) - 查看小文件全部内容**
    *   `cat /etc/os-release`: 一次性显示 `/etc/os-release` 文件的所有内容。适合看很短的配置文件。

2.  **`less` - 分页查看大文件**
    *   `less /var/log/syslog`: 查看一个可能很大的日志文件。
    *   **交互操作**:
        *   按 `↓` / `↑` 或 `PageUp` / `PageDown` 翻页。
        *   按 `/` 然后输入关键词，可以搜索。
        *   按 `q` 退出查看。

3.  **`head` 和 `tail` - 只看头和尾**
    *   `head /var/log/syslog`: 查看文件的前 10 行。
    *   `tail /var/log/syslog`: 查看文件的后 10 行。
    *   **杀手级功能**: `tail -f /var/log/syslog` - **实时监控**文件尾部的新增内容，看实时日志必备！按 `Ctrl + C` 退出。

#### **三、 系统管理和实用工具**

1.  **`sudo` (Super User Do) - 用管理员权限做事**
    *   `sudo apt update`: 我们已经用过了，凡是需要系统级权限的操作，都在前面加上 `sudo`。

2.  **`apt` (Advanced Package Tool) - 软件管家**
    *   `sudo apt install git`: 安装 `git`。
    *   `sudo apt remove git`: 卸载 `git`。
    *   `apt search git`: 搜索和 `git` 相关的软件包。

3.  **`ps` (Process Status) - 查看进程**
    *   `ps aux`: 查看当前系统上运行的所有进程。`a`=all, `u`=user, `x`=show processes without a controlling terminal。

4.  **`grep` (Global Regular Expression Print) - 文本搜索神器**
    *   `ps aux | grep jenkins`: 找出所有进程中包含 "jenkins" 关键词的行。`|` 叫做**管道符**，意思是把前一个命令的输出，作为后一个命令的输入。这是 Linux 的精髓！

5.  **`man` (Manual) - 你的随身说明书**
    *   `man ls`: 查看 `ls` 命令的详细说明书。里面有所有选项的解释。按 `q` 退出。

#### **四、 命令行小技巧 (效率倍增)**

*   **Tab 键**: **自动补全**命令或文件名，按一下或两下，能极大减少输入错误。**一定要养成习惯！**
*   **`Ctrl + C`**: 强制**中断**当前正在运行的程序。
*   **`Ctrl + L`**: **清屏**，和输入 `clear` 命令效果一样。
*   **`Ctrl + R`**: **搜索**你以前输入过的历史命令。
*   **`↑` / `↓` 箭头**: 翻阅你输入过的历史命令。

**给你的第一个练习任务：**

1.  打开终端。
2.  用 `pwd` 确认你在家目录。
3.  用 `mkdir` 创建一个名为 `my-first-dir` 的目录。
4.  用 `cd` 进入这个目录。
5.  用 `touch` 创建一个文件叫 `hello.txt`。
6.  用 `echo "Hello Linux!" > hello.txt` 命令向文件里写入一句话。（`>` 是重定向，把左边的输出写入右边的文件）
7.  用 `cat` 查看 `hello.txt` 的内容。
8.  用 `cp` 把 `hello.txt` 复制为 `hello_copy.txt`。
9.  用 `ls -lh` 查看当前目录下的文件。
10. 用 `cd ..` 返回上一级。
11. 用 `rm -r my-first-dir/` 删除你创建的目录和里面的所有文件。

完成这个小练习，你就掌握了最核心的操作。把这份教程放在手边，多用多练，很快你就会对终端操作得心应手！
