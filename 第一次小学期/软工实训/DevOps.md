
| 阶段                                   | 目的                              | 常用工具                                                                                                                                                                                                                |
| ------------------------------------ | ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. 计划与编码 (Plan & Code)**           | 需求管理、代码编写、版本控制                  | • **项目管理**：Jira, Trello<br>• **版本控制**：**Git**<br>• **代码托管**：**GitHub**, **GitLab**, Bitbucket                                                                                                                       |
| **2. 构建 (Build)**                    | 将源代码编译、打包成可执行的软件制品              | • **构建工具**：Maven, Gradle (Java), npm (Node.js), MSBuild (.NET)<br>• **容器化**：**Docker** (将应用和环境打包成镜像)                                                                                                                |
| **3. 持续集成 (Continuous Integration)** | 自动构建和运行单元测试，频繁地将代码集成到主干         | • **CI/CD 服务器**：**Jenkins** (最老牌、最灵活), **GitLab CI/CD** (与 GitLab 集成度高), **GitHub Actions** (与 GitHub 集成度高), CircleCI                                                                                               |
| **4. 测试 (Test)**                     | 自动化运行各类测试，确保软件质量                | • **单元测试**：JUnit (Java), PyTest (Python)<br>• **UI 自动化**：Selenium, Cypress, Playwright<br>• **API 测试**：Postman, Newman                                                                                              |
| **5. 制品库管理 (Artifact Repository)**   | 存储和管理构建产生的软件包（如 JAR, Docker 镜像） | • **制品库**：Nexus Repository, JFrog Artifactory, Harbor (专用于 Docker 镜像)                                                                                                                                               |
| **6. 部署 (Deploy)**                   | 将测试通过的软件制品自动部署到生产环境             | • **配置管理**：**Ansible** (简单、无代理), Puppet, Chef<br>• **基础设施即代码 (IaC)**：**Terraform** (事实标准), Pulumi<br>• **部署策略**：通过 CI/CD 工具的脚本实现蓝绿部署、金丝雀发布等                                                                         |
| **7. 运维与编排 (Operate & Orchestrate)** | 管理和调度运行环境（特别是容器）                | • **容器编排**：**Kubernetes (K8s)** (容器编排的事实标准), Docker Swarm<br>• **云平台**：AWS, Azure, Google Cloud, 阿里云等都提供了丰富的 DevOps 服务                                                                                              |
| **8. 监控与告警 (Monitor & Alert)**       | 监控系统性能、收集日志、在出现问题时发出告警          | • **监控**：**Prometheus** (指标监控) + **Grafana** (可视化)<br>• **日志**：**ELK Stack** (Elasticsearch, Logstash, Kibana) 或 **EFK Stack** (Elasticsearch, Fluentd, Kibana)<br>• **告警**：Alertmanager (配合 Prometheus), PagerDuty |
