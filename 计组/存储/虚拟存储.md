### 5.7 虚拟存储

虚拟存储是即为每个程序的存储需求**单独分配逻辑地址**，再由TLB和页表映射为物理地址访问。

考虑虚拟存储，访存的时间将分为：**虚拟地址翻译**+**物理数据读写**两个部分。虚拟地址翻译为物理地址后，将从cache开始比对

发生缺页意味着CPU需要的虚拟页在页表中没有对应的物理页，说明物理页没有被分配完或者虚拟页比物理页更多，所以在物理页被占满后还有没有对应物理页的虚拟页。

页被挪到磁盘上说明内存中的物理资源不够了，说明给进程分配的物理页超出了内存大小或者有多个进程同时在使用内存空间

*页内偏移的位数$n$决定了页的大小$N=2^n$，需要覆盖一个页中的所有字节*
*虚拟页号的位数$m$决定了虚拟页的数量$M=2^m$，也是页表项的数量*
*虚拟地址的位数即$a=m+n$*
*页表项的大小为$P$，页表的大小为$M\times P$*
*虚拟地址空间即虚拟地址可以映射到的空间，其大小为$M\times N=2^a$* 

| 物理量 | 符号/公式 | 备注 |
| :--- | :--- | :--- |
| **页内偏移位数** | $n$ | $2^n = \text{页大小}$ |
| **虚拟页号位数** | $m$ | 对应 $2^m$ 个虚拟页 |
| **虚拟地址总位数** | $a = m + n$ | CPU 发出的地址总宽度 |
| **虚拟地址空间** | $2^a$ 字节 | 程序能访问的最大理论范围 |
| **物理页号位数** | $ppn\_bits$ | 取决于实际内存条能插多大 |
| **页表项大小 $P$** | $\approx (ppn\_bits + \text{控制位}) / 8$ | **常取 4B 或 8B 以实现对齐** |
| **线性页表总大小** | $2^m \times P$ | 必须连续存放在内存中 |

#### 5.7.2 缺页失效

虚拟页的数量总是大于物理页，这样当物理页数量不足时，可以将不常用的页**转移到磁盘**Swap区上，并将磁盘地址存在没有对应物理地址的页表项上

通过定期清除引用位近似实现LRU
页表必须包含对应空间中的所有页表项，如果空间使用率很低，可以使用多级页表

增大页体积可以让TLB覆盖更大的空间，降低失效率；让页表体积减小。但是因为每个程序分配到的内存大小都是页的整数倍，大页（如给2MiB的程序分配了1GiB）会造成浪费


在数学上，计算算法的时间复杂度不仅仅是“数循环有几层”，而是要将其抽象为一个关于输入规模 $n$ 的**函数 $f(n)$（代表基本操作的执行次数）**，然后通过严格的数学极限和阶数判定，将其转化为**大O表示法（Big O Notation）**。

以下是数学上计算时间复杂度的核心理论、严格定义以及计算方法：

### 1. 数学上的严格定义

大O符号在数学上的本质是**函数渐进上界（Asymptotic Upper Bound）**。

> **数学定义：**
> 
> 设 $f(n)$ 和 $g(n)$ 是定义在正整数集上的两个正函数。如果存在正的常数 $c$ 和 $n_0$，使得对所有的 $n \ge n_0$，都有：
> 
> $$f(n) \le c \cdot g(n)$$
> 则称 $f(n)$ 当 $n$ 趋向于无穷大时是 $g(n)$ 的大O阶，记作 **$f(n) = O(g(n))$**。

用严格的**极限语言**也可以描述为：

若 $\lim_{n \to \infty} \frac{f(n)}{g(n)} = L$（其中 $0 \le L < \infty$），则 $f(n) = O(g(n))$。

### 2. 数学计算的两大基本流派

根据算法结构的不同，数学计算主要分为非递归（累加求和）**和**递归（递推方程）两大类。

#### 流派一：非递归算法 ── 级数求和 (Summation)

对于循环结构，通常需要将每层循环的执行次数写成数学上的**求和符号（$\sum$）**，然后利用级数公式求出闭式解（Closed-form expression）。

- **实例分析：**
    
    C++
    
    ```
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= i; j++) {
            // 基本操作
        }
    }
    ```
    
- **数学推导：**
    
    内层循环执行次数取决于外层的 $i$。总执行次数 $f(n)$ 可以表示为：
    
    $$f(n) = \sum_{i=1}^{n} \sum_{j=1}^{i} 1 = \sum_{i=1}^{n} i$$
    
    利用**等差数列求和公式**：
    
    $$f(n) = \frac{n(n+1)}{2} = \frac{1}{2}n^2 + \frac{1}{2}n$$
    
- **阶数判定：**
    
    计算极限 $\lim_{n \to \infty} \frac{\frac{1}{2}n^2 + \frac{1}{2}n}{n^2} = \frac{1}{2}$（常数），因此 $f(n) = O(n^2)$。
    

#### 流派二：递归算法 ── 递推关系式 (Recurrence Relations)

对于分治或递归算法，我们必须先写出它的**递推方程**，然后用数学工具求解。

##### 方法 A：主定理 (Master Theorem) ── 最快的数学公式

对于形如 $T(n) = aT(n/b) + f(n)$ 的分治递推式（将规模为 $n$ 的问题分成 $a$ 个规模为 $n/b$ 的子问题，每次合并耗时 $f(n)$），可以直接应用主定理。

通过比较 $f(n)$ 与 $n^{\log_b a}$ 的增长速度：

1. 若 $f(n) < n^{\log_b a}$，则 $T(n) = O(n^{\log_b a})$。
    
2. 若 $f(n) = n^{\log_b a}$，则 $T(n) = O(n^{\log_b a} \cdot \log n)$。
    
3. 若 $f(n) > n^{\log_b a}$，则 $T(n) = O(f(n))$。
    

- **实例（归并排序）：** $T(n) = 2T(n/2) + O(n)$
    
    这里 $a=2, b=2, f(n)=n$。计算 $n^{\log_2 2} = n^1 = n$。
    
    因为 $f(n) = n^{\log_b a}$，符合情况 2，所以直接得出 $T(n) = O(n \log n)$。
    

##### 方法 B：递归树法 (Recursion Tree) ── 几何空间求和

如果递推式两边不对称（例如 $T(n) = T(n/3) + T(2n/3) + n$），主定理失效，需要画出递归树。

- 将每一层的计算成本相加。
    
- 找出树的最大深度和每层总成本的几何级数关系，最后求和。
    

### 3. 数学简化运算的四大定理

在得出精确的 $f(n)$ 后，利用以下数学定理可以直接将其退化到大O表示法：

1. **加法规则（平行的代码块）：**
    
    如果 $f(n) = O(g(n))$ 且 $h(n) = O(k(n))$，则 $f(n) + h(n) = O(\max(g(n), k(n)))$。
    
    - _例：_ $O(n) + O(n^2) = O(n^2)$。
        
2. **乘法规则（嵌套的循环体）：**
    
    如果 $f(n) = O(g(n))$ 且 $h(n) = O(k(n))$，则 $f(n) \cdot h(n) = O(g(n) \cdot k(n))$。
    
    - _例：_ $O(n) \cdot O(\log n) = O(n \log n)$。
        
3. **等价无穷大代换（极限性质）：**
    
    当 $n \to \infty$ 时，多项式中低阶项对函数的增长速率毫无贡献。
    
    - _例：_ $f(n) = 0.001n^3 + 10000n^2 \implies O(n^3)$。
        
4. **底数不敏感定理（对数阶性质）：**
    
    在数学上，由换底公式 $\log_a n = \frac{\log_b n}{\log_b a}$ 可知，任何对数之间只差一个**常数系数 $\frac{1}{\log_b a}$**。在大O表示法中常数系数会被丢弃，因此：
    
    $$O(\log_2 n) = O(\log_{10} n) = O(\ln n) = O(\log n)$$
    

### 💡 终极思考：为什么数学上只关心 $n \to \infty$？

因为时间复杂度关注的是**渐进趋势（Asymptotic Complexity）**。在小数据量下，一个 $O(n^2)$ 的算法可能由于常数项小，运行得比 $O(n \log n)$ 还要快。但一旦 $n$ 迈向无穷大，高阶项的爆炸性增长会瞬间碾压所有低阶项和常数系数。数学计算的精髓，就是抓取那个统治全局的“最高阶龙头”。
